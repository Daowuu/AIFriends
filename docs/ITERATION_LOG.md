## Iteration 0：项目基础启动

### 本次目标

建立前后端联调骨架，让 Vue 和 Django 可以在同一个项目里协同工作。

### 已完成内容

- 建立前端工程：Vue 3 + TypeScript + Vite
- 建立后端工程：Django + DRF + SimpleJWT
- 配置 Vite 开发代理
- 配置 Django `/api/` 接口入口
- 配置 Django 托管前端打包产物
- 配置开发期 `static`、`media` 访问
- 增加根目录一键启动脚本

### 当前可用能力

- 可以分别启动前端和后端
- 可以通过根目录脚本同时启动前后端
- 前端构建产物可以打包进 Django
- Django 可以接管前端路由刷新

### 涉及模块

- 前端构建与运行
- Django 静态资源与 SPA 入口
- 本地开发代理

## Iteration 2：登录与注册模块

### 本次目标

打通用户认证闭环，让平台具备登录态、注册、退出、刷新令牌和路由守卫。

### 已完成内容

- 创建 `UserProfile` 扩展模型
- 实现登录接口
- 实现注册接口
- 实现退出接口
- 实现刷新 `access_token` 接口
- 实现获取当前用户信息接口
- 前端实现 `Pinia` 用户状态管理
- 前端实现登录页与注册页
- 前端实现登录态导航栏
- 前端实现路由守卫
- 前端实现首次打开页面时自动拉取用户信息

### 当前用户可用功能

- `/login`
  - 用户名密码登录
- `/register`
  - 用户名密码注册
- 登录成功后自动进入登录态
- 页面刷新后会尝试自动恢复登录状态
- 支持退出登录

### 当前认证设计

- `refresh_token` 存在 `HttpOnly Cookie`
- 前端内存保存 `access_token`
- 当 `access_token` 过期时，前端可用 `refresh_token` 自动刷新
- 未登录访问受保护页面时，会自动跳转到登录页

### 影响到的平台区域

- 顶部导航栏
- 登录/注册页面
- 用户状态存储
- 路由权限控制

### 当前限制

- 仅支持用户名密码登录
- 未加入验证码、邮箱验证、手机验证
- 用户资料字段还很少，只有用户名、展示名和头像基础能力

- 增加资料更新接口
- 前端实现资料编辑页
- 支持修改：
  - 用户名
  - 展示名
  - 简介
  - 头像

#### 角色模块

- 新增 `Character` 模型
- 实现角色列表接口
- 实现角色创建接口
- 实现角色详情接口
- 实现角色更新接口
- 实现角色删除接口
- 前端实现角色工作区列表页

#### 图片处理

- 支持：
  - 用户头像裁剪
  - 角色头像裁剪

### 当前用户可用功能

- `/profile`
  - 编辑个人资料
- `/workspace`
  - 查看自己的角色列表

  - 创建角色
- `/characters/:id/edit`

- 角色名
- 角色介绍
- 角色头像
- 聊天背景图
- 创建时间
- 更新时间

### 对平台意义

- 平台开始具备“用户中心”能力
- 平台开始具备“角色资产管理”能力
- 后续聊天页、角色广场、投稿流、推荐流都可以复用角色数据

### 当前限制

- 角色模块只支持“当前登录用户管理自己的角色”
- 还没有角色发布、公开广场、点赞、收藏、分享

- 还没有角色标签、开场白、示例对话、世界书等高级字段

- 支持点击角色卡片后打开聊天界面
- 聊天弹层支持：
  - 查看角色设定
  - 查看作者主页
  - 添加好友

- `/`

## Iteration 5：文字聊天与消息记录

### 本次目标

把上一轮的聊天弹层从“界面壳子”推进成“真正能发消息、能看历史、能流式回复”的可用功能。

### 已完成内容

#### 消息持久化

- 新增 `Message` 模型
- 聊天消息按好友会话持久化保存
- 每条消息记录角色、正文、创建时间
- 历史消息支持分页加载

#### 流式聊天后端

- 新增流式聊天接口
- 使用 `StreamingHttpResponse` 返回 SSE 数据
- 后端支持按块推送模型回复
- 大模型回复结束后自动写入消息记录
- 接入环境变量加载
- 增加 `backend/.env.example`
- 未配置外部模型时提供本地回退回复，保证聊天链路可直接跑通

#### 系统提示词与短期记忆

- 新增 `SystemPrompt` 模型
- 为角色聊天建立默认系统提示词
- 聊天时会自动带上：
  - 当前角色名
  - 角色设定
  - 作者信息
  - 最近 10 条对话

#### 聊天前端

- 聊天弹层接入真实历史记录接口
- 输入框接入流式发送接口
- 流式回复实时渲染到聊天记录
- 打开聊天弹层时自动聚焦输入框
- 聊天记录支持继续上滑加载更早消息
- 加载历史消息时保持当前滚动位置
- 聊天记录隐藏滚动条
- 未添加好友时输入区自动禁用并给出提示

### 当前用户可用功能

- 首页、个人主页、好友页中的聊天弹层都可以展示真实消息记录
- 已添加好友的角色可以直接发送文字消息
- 回复会逐步流式显示
- 关闭聊天弹层后再次打开，历史消息仍然存在
- 没有配置模型 API 时，仍可体验完整聊天链路

### 新增接口

- `GET /api/friend/message/history/`
- `POST /api/friend/message/chat/`

### 新增数据模型

- `Message`
- `SystemPrompt`

### 当前限制

- 当前只实现文字聊天
- 当前仍未实现 function call、长期记忆、知识库检索
- 系统提示词虽然已经入库，但还没有后台管理界面
- 当前好友关系仍然是“用户与角色”的会话关系，不是 IM 意义上的双向用户会话

## Iteration 6：模型 API 设置界面

### 本次目标

给登录用户提供一个独立的模型 API 设置界面，可以选择模型提供方、填写 API Key，并让聊天真正使用这份配置。

### 已完成内容

#### 用户级模型设置

- 新增 `UserAISettings` 模型
- 每个用户可以单独保存自己的模型配置
- 支持字段：
  - 是否启用个人配置
  - 模型提供方
  - API Key
  - API Base
  - 模型名

#### 提供方预设

- 内置阿里云百炼预设
- 内置 DeepSeek 预设
- 内置 MiniMax 预设
- 内置 OpenAI 预设
- 支持自定义兼容 OpenAI 的接口
- 当 API Base 或模型名留空时，会自动按提供方补默认值

#### 后端接口与聊天联动

- 新增模型 API 设置读取接口
- 新增模型 API 设置保存接口
- 聊天服务改为优先读取当前用户的个人模型配置
- 当个人配置关闭时，继续回退到服务端环境变量
- 当个人配置开启但未配置可用密钥时，聊天会回退本地兜底回复，保证链路不断

#### 前端设置界面

- 新增独立页面 `/settings/api`
- 支持选择提供方
- 支持填写 API Base、模型名、API Key
- 支持保留原密钥不变
- 支持显式清空已保存密钥
- 支持测试当前表单配置是否能连通模型服务
- 资料页和用户菜单新增入口

### 当前用户可用功能

- 登录后可以进入“API 设置”页面管理自己的模型配置
- 不同用户可以使用不同的模型服务
- 保存后，新发起的聊天会优先使用这份配置

### 新增接口

- `GET /api/user/settings/ai/`
- `POST /api/user/settings/ai/`
- `POST /api/user/settings/ai/test/`

### 新增数据模型

- `UserAISettings`

### 当前限制

- API Key 当前以明文形式保存在数据库中，尚未做加密存储
- 还没有为不同提供方做更细的参数面板，例如温度、max tokens、组织 ID 等

### 本轮补充

- 新增测试连接按钮
- 新增 MiniMax 提供方预设
- 支持给自己的角色自动创建会话并直接聊天

## Iteration 7：语音模块第一版

### 本次目标

在当前文字聊天基础上增加语音输入和语音回复能力，并优先把“打断上一条回复”“关闭后彻底停止监听”这类稳定性问题处理掉。

### 已完成内容

#### 消息打断与状态清理

- 当发送新消息时，会自动打断上一条还在流式生成中的回复
- 聊天框关闭时，会自动停止当前回复
- 聊天框关闭或切换角色时，会自动停止语音播放
- 语音输入模式在关闭聊天框后会自动重置回文字输入

#### 前端语音输入

- 接入 `@ricky0123/vad-web`
- 新增文字 / 语音输入切换
- 语音模式支持自动检测人声片段
- 说话时显示音浪效果
- 语音识别完成后自动把文字发到聊天后端
- 复制 `vad` 运行资源到 `frontend/public/vad/`

#### 后端语音识别

- 新增语音识别接口
- 接收前端上传的 Base64 WAV 音频
- 对接阿里云百炼 `qwen3-asr-flash`
- 当没有可用的阿里云语音配置时，返回明确错误提示

#### 语音播报

- 当前版本先使用浏览器内置 `speechSynthesis` 播报角色回复
- 只在语音输入模式下自动播放回复
- 新消息发送或关闭聊天框时，会停止旧的语音播报

#### 静态资源兼容

- 在 Django 设置里补充 `.mjs`、`.wasm`、`.onnx` 的 MIME 映射
- 避免后端运行时 VAD 资源因 MIME 类型错误导致加载失败

### 当前用户可用功能

- 聊天输入区可在文字和语音之间切换
- 语音模式下，说完一段话后会自动识别并发消息
- 语音模式下，角色回复完成后会自动播报
- 发送新消息时可以打断上一条尚未完成的回复

### 新增接口

- `POST /api/friend/message/asr/`

### 当前限制

- 当前语音识别只接了阿里云百炼 ASR
- 当前语音播报使用浏览器 TTS，不是服务端流式音频
- 还没有实现服务端 TTS 音频流和自定义音色
- 语音识别现在支持在设置页单独保存一套 ASR 配置，不再强依赖聊天模型配置
