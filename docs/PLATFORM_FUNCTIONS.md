# Platform Functions

这份文档描述 **当前版本** 的页面职责和工作流。  
它回答的是：项目现在的三条主线各自承担什么职责。

## 1. 当前产品结构

AIFriends 现在只保留三个主入口：

1. `/`
   角色列表页。
2. `/chat/:characterId`
   正式聊天页。
3. `/studio`
   角色配置、运行时配置、试聊与试听工作台。

`/workspace` 仅保留为兼容入口，会自动跳转到 `/studio`。

## 2. 首页 `/`

首页的职责只有三件事：

- 浏览角色
- 搜索角色
- 进入聊天页

首页只承担角色浏览、搜索和进入聊天的职责。

## 3. 聊天页 `/chat/:characterId`

聊天页是纯终端对话页面。

当前职责：

- 与角色进行文字聊天
- 使用语音输入
- 接收角色语音播报
- 顺序处理连续发送的多条消息
- 读取并展示该角色的持久会话历史
- 重置聊天窗口

当前聊天页不承担：

- 角色配置
- 运行时配置
- 工程态诊断

这些内容都放回 Studio。

## 4. Studio `/studio`

Studio 是唯一的配置与实验入口。

当前职责：

- 切换角色或新建角色草稿
- 编辑基础角色信息
- 编辑 `custom_prompt`
- 配置 AI 行为参数
- 配置音色
- 配置 chat / ASR / TTS runtime
- 试听音色
- 试聊角色
- 查看角色概览
- 查看最近一次实验诊断摘要
- 清空长期记忆

Studio 当前分成三块内部工作区：

1. `角色配置`
   只负责编辑角色字段、Prompt、AI 参数和语音配置。
2. `角色概览`
   用来查看角色卡片、背景图展示和当前运行时摘要。
3. `试聊诊断`
   用来试聊、试听、查看本轮调试信息和最近实验摘要。

Studio 里的工作流固定为：

1. 基础角色信息
2. 自定义 Prompt
3. AI 行为配置
4. 语音配置
5. 角色概览
6. 试聊与试听
7. 运行时配置

## 5. 当前 AI 相关产品能力

当前已经暴露到产品层的 AI 能力包括：

- 结构化 prompt 分层
- 角色自定义 prompt
- 持久会话记忆
- 语音输入
- 语音播报
- 试听与试聊
- runtime summary
- 最近一次实验诊断摘要
