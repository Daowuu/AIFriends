# Platform Functions

这份文档描述 **当前版本** 的页面职责和工作流。  
它回答的是：项目现在的几条主线各自承担什么职责。

## 1. 当前产品结构

AIFriends 现在保留四个主入口：

1. `/`
   角色列表页。
2. `/chat/:characterId`
   正式聊天页。
3. `/studio`
   角色配置、运行时配置、试聊与试听工作台。
4. `/discussion`
   多角色讨论组原型页。

`/workspace` 仅保留为兼容入口，会自动跳转到 `/studio`。

## 2. 首页 `/`

首页的职责只有三件事：

- 浏览角色
- 搜索角色
- 进入聊天页
- 跳转到讨论组实验场

首页只承担角色浏览、搜索和进入聊天/讨论入口的职责。

## 3. 聊天页 `/chat/:characterId`

聊天页是纯终端对话页面。

当前职责：

- 与角色进行文字聊天
- 使用语音输入
- 接收角色语音播报
- 独立切换“我的输入”和“角色输出”模式
- 顺序处理连续发送的多条消息
- 读取并展示该角色的持久会话历史
- 重置聊天窗口

当前聊天页不承担：

- 角色配置
- 运行时配置
- 工程态诊断

这些内容都放回 Studio。

## 4. Studio `/studio`

Studio 是唯一的配置与实验入口。

当前职责：

- 切换角色或新建角色草稿
- 编辑基础角色信息
- 编辑 `custom_prompt`
- 配置 AI 行为参数
- 配置音色
- 配置 chat / ASR / TTS runtime
- 试听音色
- 试聊角色
- 拖动角色切换条调整角色顺序
- 查看角色概览
- 清空长期记忆

Studio 当前分成三块内部工作区：

1. `角色配置`
   只负责编辑角色字段、Prompt、AI 参数和语音配置。这里还提供 `填入爱莉希雅示例` 按钮，用来一次性填入示例角色的人设、Prompt、AI 参数、音色草稿和示例素材。
2. `角色概览`
   用来查看角色卡片、背景图展示、当前运行时摘要和角色记忆。
3. `试聊诊断`
   用来试聊、试听，并查看各自的状态与输出结果。

Studio 顶部额外提供一个“讨论组实验场”快捷入口，用来进入多角色原型模式。

Studio 里的运行时配置与 `backend/.env` 保持同步，编辑后会直接回写到这份文件。

Studio 里的典型工作流是：

1. 在 `角色配置` 中编辑基础信息、Prompt、AI 参数和语音配置
2. 在 `角色概览` 中检查角色资产、记忆状态和当前运行时摘要
3. 在 `试聊诊断` 中分别进行试聊和试听
4. 如有需要，再到运行时配置区调整 chat / ASR / TTS

## 5. 当前 AI 相关产品能力

当前已经暴露到产品层的 AI 能力包括：

- 结构化 prompt 分层
- 角色自定义 prompt
- 持久会话记忆
- 语音输入
- 语音播报
- 试听与试聊
- runtime summary
- 讨论组多角色阶段原型

## 5.1 讨论组 `/discussion`

讨论组页是一个独立原型，不复用单角色聊天页。

当前职责：

- 创建多角色讨论房间
- 从角色库选择参与者
- 固定第一个角色担任主持人
- 由 DAG 驱动开场、议题拆解、角色发言、主持总结与共识收束
- 查看事件时间线和公开发言回放
- 在节点失败时暂停并手动重试
- 重置房间并重新开始
